# 可筛选 3D 血缘图规范

## ADDED Requirements

### Requirement: 文档结果必须支持类型筛选

事件列表 MUST 支持按事件类别筛选，实体列表 MUST 支持按实体类型筛选；类型筛选 MUST 与标题关键字同时生效，并驱动匹配数量和分页。

#### Scenario: 筛选字段级血缘事件

- **WHEN** 用户选择字段到字段事件类别
- **THEN** 列表只显示该类别事件
- **AND** 每条事件显示其事件类别

### Requirement: 图谱必须使用可交互 3D 布局

类型化血缘图 MUST 使用白色画布上的 Three.js/WebGL 三维布局，并支持旋转、缩放、点击聚焦、实体详情、搜索、一跳展开、复位和适配视图。

#### Scenario: 打开大型类型化项目

- **WHEN** 用户进入图谱页
- **THEN** 3D 模块按需加载并渲染受上限约束的骨架
- **AND** 画布背景保持白色，节点、边和标签具有足够对比度
- **AND** 不预取几万字段的完整图谱

### Requirement: 实体与关系必须可独立显隐

任务、表、字段节点和各自名称标签 MUST 能独立显隐；任务-任务、任务-表、表-表、表-字段、字段-字段关系 MUST 能分别显隐。

#### Scenario: 只查看表级链路

- **WHEN** 用户隐藏任务和字段节点并只开启表-表关系
- **THEN** 画布只保留表节点及表之间的数据流或连接关系
- **AND** 重新开启筛选时无需重新请求已加载数据

### Requirement: 任务依赖必须从类型化关系推导

系统 MUST 从生产任务、来源表和消费任务上下文推导任务-任务依赖，MUST NOT 为此增加任务到来源表的错误直连。

#### Scenario: 上游任务产出消费表

- **WHEN** 任务 A 产出表 X 且任务 B 通过表 X 产出下游表
- **THEN** 图谱返回 `A -> DEPENDS_ON -> B`
- **AND** 保留 `A -> PRODUCES -> X` 及表级数据流

### Requirement: 类型必须具有稳定视觉编码

任务、表、字段 MUST 使用三种稳定且可区分的节点颜色；五种关系类别 MUST 使用稳定颜色并在筛选面板提供图例。

#### Scenario: 切换实体类型

- **WHEN** 用户隐藏或恢复某类实体
- **THEN** 图例颜色与画布节点颜色保持一致
