# Proposal: 分层语义血缘图改造

## 背景

当前类型化血缘图使用 3D 力导向球体和空间连线。该视图可以展示拓扑规模，但表名、字段归属、加工方向和关系类型难以连续扫描，字段量增加后遮挡和空间漂移进一步降低可读性。

GitNexus 的可取之处不只是力导向图，而是“资源浏览器 + 图谱画布 + 检查器”的工作台信息架构。SQL 血缘具有明确方向、字段归属和加工语义，因此本变更借鉴其工作台结构，但不照搬适合通用代码概览的无方向力导向布局。

## 目标

- 将类型化血缘默认画布改为白底 2D 左右分层 DAG，上游位于左侧、下游位于右侧。
- 使用任务、表和字段的语义化节点 UI；字段通过 `HAS_COLUMN` 关系收进所属表卡片。
- 使用带方向箭头、关系类型和选中状态的可读连线。
- 将图谱页组织为显示控制、中央画布和血缘检查器三个协同区域，避免浮层长期遮挡图谱。
- 保留搜索、类型筛选、关系筛选、`1-5` 层穿透、增量加载、复位和详情能力。
- 在桌面和移动视口内保持可操作，并继续遵守现有前端节点与请求上限。

## 非目标

- 不修改结构化血缘协议、摄取逻辑、关系存储或搜索算法。
- 不在本变更中补做 SQL 文件监听、增量索引、源码读取或通用 Cypher 查询；这些属于 SQL CodeGraph 索引内核的后续变更。
- 不过滤临时表或其他已摄取实体。
- 不在首屏展开全部字段或突破现有增量加载上限。
- 不保留 3D 视图作为类型化血缘的默认模式。

## 影响

- 类型化血缘画布由 Three.js 组件迁移到现有 React Flow 体系，并增加 ELK 分层布局。
- 新增语义画布视图模型、自定义任务/表/字段节点和自定义关系边。
- 图谱工作区改为 GitNexus 风格的三段式检查工作台，但继续沿用 SAG 紧凑、白底的设计语言。
- 删除仅被类型化血缘使用的 3D 运行依赖。
- 更新视觉验收脚本、设计基线和用户文档。
