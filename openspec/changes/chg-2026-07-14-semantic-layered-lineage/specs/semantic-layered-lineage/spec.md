# 分层语义血缘图规范

## REMOVED Requirements

### Requirement: 图谱必须使用可交互 3D 布局

**Reason**：3D 力导向布局无法稳定表达 SQL 血缘的左右方向、字段归属和关系语义。

**Migration**：类型化血缘迁移到 React Flow + ELK 的白底 2D 分层布局；普通事件-实体图保持不变。

## ADDED Requirements

### Requirement: 类型化血缘必须使用分层语义布局

类型化血缘图 MUST 使用白色画布上的 2D 分层布局，MUST 将上游放在目标实体左侧、下游放在右侧，并以稳定位置和方向箭头表达数据流。

#### Scenario: 查看表级上下游

- **WHEN** 用户打开一张具有上下游关系的表
- **THEN** 上游实体显示在其左侧且下游实体显示在其右侧
- **AND** 关系箭头明确指向目标实体

### Requirement: 字段必须收进所属表卡片

系统 MUST 使用 `HAS_COLUMN` 关系确定字段所属表，并将字段显示为表卡片内的字段行；没有归属关系的字段 MUST 仍以独立字段节点显示。

#### Scenario: 展开字段血缘

- **WHEN** 一张表存在字段血缘且用户展开字段列表
- **THEN** 字段显示在该表卡片内
- **AND** 字段关系连接到对应字段行端口

#### Scenario: 字段没有归属表

- **WHEN** 一个字段没有任何 `HAS_COLUMN` 入边
- **THEN** 画布仍显示该字段及其关系
- **AND** 不通过名称猜测虚构所属表

### Requirement: 关系必须表达方向和业务类型

所有可见关系 MUST 有目标箭头；表级连接事件 MUST 直接显示 `LEFT JOIN`、`RIGHT JOIN`、`FULL OUTER JOIN`、`INNER JOIN` 等关系类型，其他关系 MUST 在选中或开启标签时显示可理解的语义标签。

#### Scenario: 表通过左连接参与加工

- **WHEN** 血缘关系类型为 `LEFT_JOIN`
- **THEN** 关系标签显示 `LEFT JOIN`
- **AND** 点击关系可打开对应事件详情

### Requirement: 语义画布必须保留现有探索能力

搜索、实体与关系筛选、节点增量加载、`1-5` 层穿透、详情、复位和适配视图 MUST 在 2D 画布中继续可用。

#### Scenario: 两层穿透

- **WHEN** 用户选中实体并将穿透层级从 1 调整到 2
- **THEN** 画布按需加载并显示两跳内的上下游语义节点和关系
- **AND** 无关关系保持隐藏

### Requirement: 图谱页必须提供协同检查工作台

桌面端类型化血缘页 MUST 提供显示控制、中央画布和血缘检查器三个协同区域；持续可见的控制或详情区域 MUST NOT 覆盖中央图谱。窄视口 MUST 优先保留画布，并允许按需打开控制与详情。

#### Scenario: 检查选中表

- **WHEN** 用户在桌面端选择一张表
- **THEN** 中央画布只强调穿透邻域
- **AND** 右侧检查器显示表名称、关系证据和穿透规模
- **AND** 左侧控制区与检查器均不遮挡图谱节点

### Requirement: 语义画布必须渐进展示字段

表卡片 MUST 限制首屏字段行数量，MUST 提供展开和收起操作，且选中的字段 MUST 始终保持可见。

#### Scenario: 表包含大量字段

- **WHEN** 一张表包含超过默认展示数量的字段
- **THEN** 卡片只显示有限字段行和剩余数量
- **AND** 用户可以展开更多字段而不触发重新摄取
