# DESIGN.md

## 设计基线

1. WebUI 延续 SAG 现有的紧凑工作台布局、灰阶视觉、组件尺寸和交互模式。
2. 结构化事件扩展默认不新增页面；事件与实体继续复用现有文档、图谱和检索视图。
3. `task`、`table`、`column` 类型应直接显示真实名称，不使用无意义别名或装饰性前缀。
4. 若后续增加类型筛选或视觉区分，必须保持普通文档实体类型兼容并覆盖桌面与移动视口。
5. Ant Design 相关改动必须同时遵循工作区 `root_docs/design/ant-design/` 规范。
6. 类型化血缘图使用按需加载的 React Flow + ELK 白底二维分层 DAG；任务、表、字段分别使用适合白底的琥珀、蓝色、绿色，普通事件-实体图继续使用原二维组件。
7. 字段通过显式 `HAS_COLUMN` 关系收进所属表卡片；无归属字段保留独立节点。表级 JOIN 直接显示 SQL 关系类型，其他关系标签按需显示。
8. 点击实体后只布局并显示当前 `1-5` 层穿透邻域；取消选择后恢复完整已加载图，不能因聚焦大图而把目标链路缩成不可读的缩略图。
9. 类型化血缘桌面端采用显示控制、中央画布和血缘检查器三段式工作台；持久面板不得覆盖画布，窄视口优先保证画布操作。
