# Proposal: 可筛选的 3D 血缘图谱

## 背景

文档事件和实体只能按标题搜索，无法按 SQL 事件类别或实体类型收窄结果。现有二维图谱在任务、表、字段和关系同时出现时容易产生标签重叠，也没有按实体类型、标签和关系类别独立显隐的能力。

## 目标

- 文档事件支持按事件类别筛选，实体支持按实体类型筛选。
- 血缘图使用白色画布上的 Three.js/WebGL 3D 力导向布局，保持搜索和一跳增量加载。
- 任务、表、字段采用稳定且不同的颜色，并可分别控制节点和名称标签显隐。
- 任务-任务、任务-表、表-表、表-字段、字段-字段五类关系可分别控制显隐。
- 从任务产出和表数据流中推导任务依赖，不把任务与来源表错误直连。
- 桌面和移动端都能访问筛选、复位、聚焦和详情操作。

## 非目标

- 不加载完整项目图谱到浏览器。
- 不改变普通文档的抽取流程和实体类型。
- 不把 3D 坐标持久化到数据库。

## 影响

- 事件 API 增加可选 `category` 字段。
- 类型化图 API 增加由既有关系推导的 `DEPENDS_ON` 任务关系。
- 血缘图前端从 React Flow 切换为按需加载的 `react-force-graph-3d`。
- 文档结果栏和图谱工具栏增加筛选状态。
