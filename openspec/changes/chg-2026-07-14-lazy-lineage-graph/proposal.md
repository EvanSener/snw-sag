# Proposal: 可扩展的任务-表-字段血缘图

## 背景

SAG 上游图谱是事件与实体的二部图，项目工作区加载时会一次查询并传输全部实体、事件和边。该模型既无法区分任务产出与事件上下文，也无法承载几千任务、几万字段的交互式图谱。

## 目标

- 兼容摄取 `snw.sql_lineage_event.v2` 类型化关系。
- 持久化任务、表、字段之间的有向关系和任务上下文。
- 首屏只加载任务到产出表骨架和可推导的任务依赖，节点点击时按一跳邻居增量加载。
- 提供实体搜索、单次结果上限、前端累计节点上限和可继续展开提示。
- v1 信封和普通 SAG 文档继续兼容。

## 非目标

- 不删除上游事件-实体图。
- 不在浏览器执行全图布局或全图遍历。
- 不从普通文档猜测 SQL 类型关系。

## 影响

- 新增 `lineage_relations` 数据表、索引和 API。
- ingestion 在同一事务中写入实体、事件和类型关系。
- 图谱页优先展示类型化血缘图；没有类型关系的项目回退上游图。
