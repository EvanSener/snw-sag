# Design: 可筛选的 3D 血缘图谱

## 文档筛选

事件列表从 `events.category` 读取事件类别；空类别归入“未分类”。实体列表继续使用 `entities.type`。筛选在当前文档已加载结果上执行，并与标题关键字取交集；分页始终基于筛选后的集合。

## 任务依赖

任务依赖不新增来源表直连。若任务 A 通过 `PRODUCES` 产出表 X，表 X 又通过带任务 B 上下文的 `DATA_FLOW` 参与任务 B 的目标表生产，则图 API 推导 `A -> DEPENDS_ON -> B`。相同任务的自依赖被忽略，重复路径按任务对聚合证据数。

## 3D 渲染

`react-force-graph-3d` 使用 Three.js/WebGL 和三维力导向布局。主画布使用白色背景，全宽且具有稳定高度，通过 `ResizeObserver` 同步尺寸。工具栏、筛选面板和详情面板使用同一浅色工作台视觉。图数据仍由初始骨架、实体搜索和节点一跳展开增量合并，前端上限维持 500 个节点。

任务、表、字段分别使用适合白底的中深琥珀、蓝色和绿色；节点大小由关系数量的对数函数约束。边按端点类型归入五种关系类别并使用独立的中深色，方向箭头表达流向。标签使用白底深色文字，默认只显示任务，表和字段标签按需开启。

## 显隐控制

实体控制分成两个维度：

- 节点：任务、表、字段各自显示或隐藏。
- 标签：任务、表、字段名称各自显示或隐藏。

关系控制按任务-任务、任务-表、表-表、表-字段、字段-字段分别显示或隐藏，并提供独立的关系标签总开关。隐藏节点时，其关联边同时隐藏；隐藏关系不会删除已加载数据，重新开启立即恢复。

## 性能与兼容

3D 模块使用动态 import 独立分包，只有进入图谱页才下载。画布不渲染不可见节点和边，字段标签默认关闭。没有类型化关系的项目继续使用上游二维事件-实体图。
