# Proposal: 血缘实体穿透高亮

## 背景

当前点击血缘实体只聚焦相机并加载一跳，画布仍同时展示全部关系。用户无法区分与选中实体直接或间接相关的链路，也无法控制需要向上下游穿透多少层。

## 目标

- 点击实体后只高亮与它直接或间接相关的关系，无关节点保留为弱化上下文。
- 提供 `1-5` 层穿透控制，切换层级时按需加载对应邻域。
- 同时覆盖上游和下游关系，并继续遵守实体类型与关系类型筛选。
- 搜索、重置、详情、移动端和白色 3D 画布继续兼容。

## 非目标

- 不改变或过滤已摄取的任务、表、字段和关系。
- 不把穿透结果持久化为新的血缘关系。
- 不突破现有 500 个前端可见节点上限。

## 影响

- 前端图模型增加多跳邻域计算。
- 选中实体面板增加穿透层级步进控制。
- 3D 节点和关系视觉根据当前选中邻域动态高亮。
- 节点一跳 API 由受限的广度加载器按层复用。
